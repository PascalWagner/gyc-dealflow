Internal notes for picking up where we left off. Last updated: March 16, 2026.

Working

  • ✓ Sign In via email+password (GHL contact lookup via Make.com webhook)
  • ✓ Sign Up creates GHL contact with dealflow-user tag (via highlevel:createAContact module)
  • ✓ Duplicate sign-up returns friendly error
  • ✓ Auth gate on dashboard (redirects to login if no valid session)
  • ✓ Logout clears localStorage session
  • ✓ Live deal data rendered from embedded JSON (~30+ deals)
  • ✓ Deal filtering, search, grid/list views
  • ✓ Buy Box builder & scoring
  • ✓ GP directory with follow/unfollow
  • ✓ Deployed to dealflow.growyourcashflow.io via Vercel
  • ✓ Code on GitHub: git@github.com:PascalWagner/gyc-dealflow.git

Not Working / Known Issues

  • ✗ Forgot password flow (webhook not built)

  • ✗ GHL highlevel:universal POST body broken (sends [object Object]) — use dedicated modules like createAContact instead
  • ✗ Session token is a timestamp, not cryptographically secure

Lessons Learned

  • Make.com Data Store modules created via API have a bug — field interfaces don't resolve, outputs are always empty. Must use the UI to configure data store modules.
  • Make.com highlevel:universal module with body type any serializes objects as [object Object] instead of JSON. Use dedicated GHL modules (createAContact, searchContacts) instead.
  • first(array).property doesn't work in Make.com templates. Use get(first(array); "property") instead.
  • Webhook UDT must be set as a data structure integer ID, not an inline array spec.
  • Make.com if() with escaped quotes in JSON strings produces garbled output. Keep response bodies simple.
